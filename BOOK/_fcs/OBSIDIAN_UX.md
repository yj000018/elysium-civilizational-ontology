# Obsidian UX

Obsidian is the human operator interface for FCS. It is not the source of truth. The source of truth remains the Markdown/YAML files, manifests, scripts, and GitHub repository.

## Dashboards
Dashboards are Dataview-friendly documents located in `BOOK/views/dashboards/`. They provide real-time visibility into structure, actions, reviews, and resources.

## Plugin Stack
Required/recommended plugins include Dataview, Templater, QuickAdd, Graph View / Local Graph, and Canvas.

## Workflow
The founder uses Obsidian to navigate the Master Content Layer, view dashboards, trigger QuickAdd/Templater commands to generate action requests, and review rendered outputs. Direct editing of prose is minimal, focusing instead on AI command generation and strategic review.
