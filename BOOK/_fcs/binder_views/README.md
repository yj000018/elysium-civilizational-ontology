# FCS Binder Views — README

## Core Principle

**`binder_view.md` files are generated snapshots, not source of truth.**

Never edit generated binder files directly to change manuscript structure.

---

## File Map

| File | Type | Description |
|---|---|---|
| `OBSIDIAN_DYNAMIC_BINDER.md` | Dynamic (Dataview) | Live full-manuscript binder in Obsidian |
| `OPENING_DYNAMIC_BINDER.md` | Dynamic (Dataview) | Live Opening Part binder in Obsidian |
| `current_binder_view.md` | Generated snapshot | Full manuscript export (regenerated by script) |
| `opening_binder_view.md` | Generated snapshot | Opening Part export (regenerated by script) |
| `PHASE_III_1A_R_OPENING_BINDER_VIEW.md` | Static (archived) | Phase III-1A-R extraction snapshot |
| `PHASE_III_1A_S1_OPENING_MACRO_BINDER_VIEW.md` | Generated snapshot | Phase III-1A-S1 macro structure snapshot |

---

## Source of Truth

Edit these, not the generated files:

1. **Module frontmatter** — `BOOK/manuscript/**/*.md`
2. **Central registry** — `BOOK/_fcs/registries/book_module_registry.yaml`
3. **Part manifests** — `BOOK/manuscript/<part>/manifest.md`

---

## Regenerate Static Exports

After any structural change:

```bash
python scripts/generate_binder_view.py
```

---

## Obsidian Dynamic Binder

Requires: **Obsidian Dataview plugin**

Open `OBSIDIAN_DYNAMIC_BINDER.md` or `OPENING_DYNAMIC_BINDER.md` in Obsidian for live views.
