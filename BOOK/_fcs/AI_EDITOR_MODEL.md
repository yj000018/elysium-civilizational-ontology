# AI-First Editor Model

FCS is AI-command-first.

The founder edits primarily through:
- prompts
- command palette actions
- action request files
- node/module context commands
- dashboard-driven operations

Direct human prose editing is secondary and mostly reserved for late-stage cleanup after FCS output is transferred into downstream tools.

## Supported Actions

The AI-first editor model must support:
- write
- develop
- expand
- shorten
- densify
- clarify
- restructure
- split
- merge
- move
- review
- check
- extract resources
- generate summaries
- render views
- produce handoff outputs
