## 6. Current Implicit Opening Sequence

The Manus-drafted sequence follows a linear diagnostic-to-map progression:

1. **Diagnose** — Name the polycrisis as a single systemic pathology
2. **Demolish** — Show why siloed expertise fails
3. **Reframe** — Introduce the metabolic metaphor as the new lens
4. **Map** — Reveal the Seven Foundations as the book's architecture

This is a valid intellectual progression but may be too linear, too fast, and insufficiently layered for a book opening that needs to:
- Hook emotionally
- Build trust
- Create intellectual desire
- Establish voice
- Promise transformation

## 7. Structural Problems Detected

1. **Writing started before ultra-fine TOC validation** — modules were drafted from a coarse 4-module plan without submodule/beat-level architecture
2. **Seven Foundations revealed too early** — Module 04 reveals the full architecture in the opening, which may reduce anticipation for Part 02
3. **Module granularity too coarse** — 4 modules × ~1500 words each is a flat structure; a book opening likely needs more hierarchical layering (movements → chapters → modules → beats)
4. **No drill-down hierarchy** — no movements, no submodules, no beats defined
5. **No executive summaries per module** — no clear statement of what each module must accomplish before writing
6. **Lacks binder-level architecture** — no intermediate structure between "Part 01" and "4 modules"
7. **No writing briefs** — modules were written without formal writing briefs specifying voice, pacing, reader state transitions
8. **Tone not validated** — voice/style not tested against BOOK_STYLE_AND_VOICE_GUIDE before committing to prose
9. **No source protocol** — all modules note SOURCE_USAGE_PROTOCOL_NEEDED_BEFORE_PUBLICATION
10. **Routing violation** — Manus wrote prose that should have been routed through ChatGPT (structure) + Claude (prose)
