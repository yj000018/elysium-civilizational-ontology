## 1. Purpose

The Publication / Release Readiness Protocol defines when an artifact, document, book section, protocol, website, dataset, or package is ready to be released, published, shared publicly, or used as downstream source.

Core principle: Release requires a higher gate than internal completion.

---

## 2. Release Status Labels

NOT_READY, INTERNAL_DRAFT, REVIEW_PENDING, QA_PENDING, FOUNDER_APPROVAL_PENDING, CLAUDE_REVIEW_PENDING, RELEASE_CANDIDATE, APPROVED_FOR_INTERNAL_USE, APPROVED_FOR_PUBLIC_RELEASE, RELEASED, RELEASE_REJECTED, RELEASE_SUPERSEDED, WITHDRAWN.

---

## 3. Release Readiness Gates

Before release, check: source of truth is clear, canonical/candidate status is correct, artifact QA passed, required reviews complete, Founder approval obtained if needed, Claude review complete if required, Git tag/commit exists if relevant, Notion status correct if relevant, Mem0 durable fact updated if relevant, no unresolved blockers, no prohibited claims, and next downstream use is clear.
